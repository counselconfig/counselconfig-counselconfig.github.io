1.    Introduction

 

1.1        About the business

 

The Queens Printer at The National Archives (TNA) publishes legislation of the United Kingdom (UK) as Parliament enacts. TNA delivers the official online database of publicised UK statute law documents at www.legislation.gov.uk underpinned by an Application Programming Interface (API).

 

1.2        Problem statement

 

Conditional statements containing disproportionately high quantities of logical connectives in legislation makes it difficult to understand (Genesereth, 2015). This is a source of latent complexity that cannot be detected only until a user attempts to read the legislation. Natural language processing (NLP) is proposed to train a model to extract logical connectives for text analysis. This will allow a user to be well-informed about the interpretable friendliness of a piece of UK legislation before reading it.

2.2        Justification for selected techniques

 

Reasons for the selected techniques are given in Table 2.2.

 

 

Table 2.2: Selected techniques

Techniques

Reason for selection

 

Classification

 

Word like and or but need to be extracted and classified as logical connectives for analytics and hypotheses testing. It is a sequencing labelling problem in supervised machine learning attracting the NLP solution Named-Entity Recognition and Classification (Ji, et al., 2019). Often simply NER (Perera, et al., 2020) it is associated with classifying real-world objects of interest represented by a noun (Ahmad, et al., 2020). NER is customisable and a key information extraction subtask (Zia & Sharif, 2019). NER can be used to train a model to recognise key words defined by language L1 to be represented by their symbolic logical connective. Many NLP libraries exist for custom training NER (Gudivada & Arbabifard, 2018)

 

Text Analysis

The process to extract proper text patterns from the unstructured text data, which are collected from different written resources is text analysis (Khan, et al., 2020). This is appropriate given data resources will be different unstructured text documents.   The scope of text analysis also covers: machine learning, statistics, and linguistics (Li & Chen, 2018) - all exercisable in pursuit of the project analytical goals. Text analysis applicability appears in sub-techniques (Li, et al., 2018) among which information extraction is most pertinent to recognising logical connectives in statutes. Text data will need to be obtained, cleaned, and read (normalisation). There are various methods to improve text transformation to canonical format (Faldu & Atkotiya, 2019).

 

 

 

 

2.3        Justification for rejected techniques

 

Among the two candidate techniques selected four were rejected for the reasons explained in Table 2.3.

 

 

Table 2.3: Rejected techniques

Techniques

Reason for rejection

 

Association Rules

 

Machine learning association rules are often used in marketing to leverage buying pattern relationships between data attributes X,Y  described as if-then statements X → Y (Yahyaoui, et al., 2020). However, this is inconsistent with the goals of the study, which seek to train a model to identify and classify entities. The possibility that certain word combinations could determine the probability of a logical connective would not reveal the actual entity being sought.

 

Clustering

Revealing hidden structures in unlabelled data for categorisation defines an unsupervised learning clustering algorithm used in industries like healthcare and banking (Dutta, 2018). To the contrary, this project needs to classify certain attributes of data with symbolic logical operators. The learning needs to be supervised with labelled examples in a statute dataset for training where each element is a feature vector (Burkov, 2019).

 

Regression

Often used in real estate with independent parameters like rooms, area &c, regression learning algorithms can be used to solve the problem of forecasting a target numerical or continuous value like price (Kuvalekar, et al., 2020). This differs from the classification problem which this project aims to solve through multinominal means where the value sought is a discreet value (Muthukumar, et al., 2020) such as P, P ∧ Q, and so on, which are members of a finite set of classes in language L1.   

 

Time Series Analysis

Unlike regression, predicting a value based on previous observed sequential measurements is an analysis of time series data found in training stock price forecasts  (Navon & Keller, 2017). This learning approach is inapplicable to legislation corpus in which text data is to be labelled to produce a model. Whilst legislation documents are enacted at specific dates, these data are not a series of indexed points that can be used to train a deduction of required text labels.

Once the component ruler = nlp.create_pipe("entity_ruler") was implemented in Build it was ready to be added to the spaCy pipeline in the training stage to initiate a novel NER pipe. The pipeline was responsible for driving the logical connective predictions grounded on current value weightings of the model i.e., gradient. These weight values were derived from estimates of the labelled examples in text data provided for the training.  In effect, the pipeline took LPA (1925) data and performed a series of operations on it through each pipe before outputting new data. This was possible by calling nlp(), which initiates a tokeniser pipe retuning a Doc container to access annotations which is handed down further pipes as figure shows. In this project a Doc.ents object was produced for training the NER model Figure 3.3.

4.2        Conclusion

 

This report provided a solution for finding logical complexity in UK legislation with a bespoke NER model. The business goals were mostly met save for insufficient time iterating to standard. Three statutes under hypotheses were not significantly different but testing more groups might show otherwise. Operationalisation would require more supervision to avoid overfitting or noise from spurious words (Wang, et al., 2020). This might be remedial with extra statute training data and relation extraction retrieval rules (Bruches, et al., 2020).
